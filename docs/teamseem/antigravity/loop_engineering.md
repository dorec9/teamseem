## 🧭 1. 제어 패러다임의 진화 경로

```
프롬프트 엔지니어링 → 컨텍스트 엔지니어링 → 하네스 엔지니어링 → 루프 엔지니어링
(단일 질의 최적화)   (정보 환경 전체 설계)   (실행 인프라+제약)     (자율 순환 제어)
```

| 단계 | 핵심 동작 | 인간의 역할 |
|---|---|---|
| **프롬프트 엔지니어링** | 프레이징과 예시를 수동 미세 조정 | 질문 작성자 |
| **컨텍스트 엔지니어링** | 컨텍스트 윈도우에 최적의 토큰 세트 배치 설계 | 정보 설계자 |
| **하네스 엔지니어링** | 모델 + 실행 인프라 + 제약 장치 + 도구 바인딩 | 시스템 설계자 |
| **루프 엔지니어링** | 목적 정의 → 시스템이 자율 순환 기동 | **지휘자/아키텍트** |

> **핵심 공식**: `에이전트 = 모델 + 하네스` (Mitchell Hashimoto)

## 🔄 2. 에이전틱 루프의 이중 레이어 구조

### 내부 루프 (Inner Loop)
`인식(Perceive) → 추론(Reason) → 계획(Plan) → 행동(Act) → 관찰(Observe) → 교정`

### 외부 루프 (Outer Loop)
스케줄러, 이벤트 트리거, 외부 평가 시스템이 작업을 공급하고 최종 정합성을 통제.

### 열린 루프 vs 닫힌 루프

| 차원 | 열린 루프 (Open Loop) | 닫힌 루프 (Closed Loop) |
|---|---|---|
| **동작 원리** | 에이전트가 자유 탐색 | 정량적 성공 기준(Verifier)으로 매 단계 검증 |
| **장점** | 창의적 우회 해결책 발견 | 비용 예측 가능, 일관된 품질 보증 |
| **리스크** | 무한 루프, 자원 폭발 | 창의성 억제, 검증 기준 완성도에 종속 |
| **적합 분야** | R&D 프로토타이핑 | **전력망 발전량 예측**, 정량 제어 계통 |

## 🏗️ 3. 오케스트레이션 워크플로우 패턴

| 패턴 | 메커니즘 | 유스케이스 |
|---|---|---|
| **프롬프트 체이닝** | 이전 출력 → 다음 입력으로 순차 바인딩 | 단계별 기술 문서 작성 |
| **병렬 처리** | 독립 영역 분사 → 동시 연산 → 통합/투표 | 동시 다발적 보안 스캔 |
| **평가자-최적화자** | Generator → Evaluator → 재시도 강제 | 벤치마크 기반 최적화 |
| **오케스트레이터-워커** | 마스터가 동적 분할 → 워커에게 배분 취합 | 대규모 모듈 통합 |

## 🔧 4. 루프 엔지니어링의 6대 구성 요소

1. **오토메이션 (Automations)**: Cron 스케줄, CI/CD 이벤트 트리거로 루프 자동 기상
2. **워크트리 (Worktrees)**: Git Worktree 기반 다중 에이전트 격리 샌드박스
3. **스킬 (Skills)**: SKILL.md에 팀 규칙, 불변 규칙, 빌드 지침을 영속 기록
4. **플러그인/커넥터**: MCP 서버 경유 외부 자산(Jira, DB, API) 연동
5. **서브에이전트**: 생성 에이전트와 분리된 검사역 서브에이전트로 교차 검증
6. **영속성 상태 저장소**: 디스크 기반 물리 메모리로 컨텍스트 윈도우 폭발 방지

## 🧪 5. xIL(X-in-the-Loop) 검증 아키텍처

가상 → 실물 하드웨어로 점진적으로 확장되는 단계적 검증 체계:

`MIL → SIL → PIL → FIL → HIL → VIL`

- **MIL/SIL**: 완전 가상화, 알고리즘 논리 검증에 최적
- **HIL**: 실제 하드웨어와 폐루프 결선, 타이밍/네트워크 지연/메모리 한계 탐지
- **풍력 발전 맥락**: AI 예측 모델(SIL) → 실제 SCADA 데이터 기반 백테스트(HIL 유사)

## 👥 6. 인간 참여 수준별 분류

| 유형 | 인간의 역할 | 적용 상황 |
|---|---|---|
| **HITL (Human-in-the-Loop)** | 루프 중간에 최종 승인 필수 | 안전 필수 업무, 재무적 손실 위험 |
| **HOTL (Human-on-the-Loop)** | 실시간 원격 관제, 이상 시 수동 오버라이드 | 자율 운영 + 감독 체계 |
| **Human-outside-the-Loop** | 완전 자율 | 저위험 반복 태스크 |

## 🏭 7. SASE(SE 3.0) — 구조화된 에이전틱 소프트웨어 공학

### 이중 워크벤치
- **ACE (Agent Command Environment)**: 인간용 관제 — 인텐트 정의, 모니터링, 병합 검토
- **AEE (Agent Execution Environment)**: 에이전트 전용 무인 샌드박스 — 파일 생성, 컴파일, 격리 테스트

### 3대 명세 규격
1. **브리핑 스크립트 (BriefingScript)**: 기능 명세 + 불변 조건 + 환각 방지 바인딩 + 검증 루프 명세
2. **루프 스크립트 (LoopScript)**: 아키텍처 흐름 + SOP + Task Decomposition + 병렬 오케스트레이션
3. **멘토 스크립트 (MentorScript)**: 암묵적 코딩 규범 + 금지 패턴 + 복잡도 지표 한계

### MRP (Merge-Readiness Pack)
$$MRP = \{F_{completeness}, S_{verification}, E_{hygiene}, C_{rationale}, A_{trail}\}$$
